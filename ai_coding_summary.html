📋 项目概览

《海洋冒险》是一个完全在浏览器中运行的HTML5小游戏。玩家控制一条小鱼，在60秒内不断吞噬比自己小的鱼来进化， 从小虾米开始，历经10个阶段成长为海洋霸主。项目采用AI辅助编程的方式完成， 从最初的代码编写、BUG修复、到后期部署、跨设备排行榜，全程由AI工具（WorkBuddy + Deepseek）辅助实现。

🚀 开发经历

Phase 1 · 框架搭建

用文本描述生成了完整游戏框架

直接告诉AI "做一个海洋吞噬进化的游戏"，AI就生成了完整的Canvas游戏框架——包括鱼类的移动、碰撞检测、进化系统、AI对手等核心逻辑。

Phase 2 · 界面美化

AI生成写实海底精灵贴图

利用AI绘图工具生成了10种海洋生物的写实精灵贴图，并编写程序自动去除背景、生成精灵动画帧。

Phase 3 · Bug围剿

游戏逻辑修复与性能优化

修复了NPC鱼游不动、进化进度显示错误、音效冲突等一系列问题。最经典的BUG是变量命名冲突导致的NaN错误——"sp"同时被用作速度和精灵名称。

Phase 4 · 部署上线

GitHub Pages + CDN 发布

通过GitHub Pages部署到公网，配置jsDelivr CDN加速国内访问，添加OG卡片实现微信分享。

Phase 5 · 跨设备同步

排行榜从本地走向云端

先后尝试了GitHub Actions自动同步、Issues API跨域写入等多种方案，最终采用Supabase数据库实现稳定、实时的排行榜同步。

💡 AI编程心得

核心感悟

"AI编程不是让AI取代程序员，而是让一个人拥有一个全栈团队的能力。 从需求分析、代码实现、UI设计、运维部署到问题排查，AI都能参与。 关键在于——你知道该问什么，以及如何判断AI给出的答案是否靠谱。"

1. 描述能力就是编程能力

这次体会最深的是：能把需求描述得越清晰，AI输出的代码质量就越高。 写自然语言需求文档的能力，正在变成一种新的编程能力。 与其让AI猜你想做什么，不如把"怎么做"拆解清楚。例如：

  • "做一个海洋游戏" → 得到泛泛的框架
  • "做一个Canvas 2D游戏，玩家滑动控制一条鱼，60秒倒计时，吞噬比自己小的鱼进化，有10个进化阶段" → 得到一个可用的游戏

2. BUG修复是最能体现AI价值的场景

遇到诡异的BUG（比如所有NPC鱼都变成NaN），自己逐行排查可能需要半小时甚至更久，但AI可以：

  • 快速定位问题根因（数据流追踪）
  • 提供多种修复方案并分析利弊
  • 一边修复一边解释原理，顺便学到新东西

这次遇到的"NaN鱼群"BUG，根源是一个数组中的字段名冲突（sp被重复赋值），AI从报错信息直指代码行，5分钟内修复。

3. 架构决策 AI 也能参与

跨设备排行榜是项目中最大的技术挑战。AI帮我分析了几种方案：

  • ❌ GitHub Actions触发写入 — CORS限制，浏览器无法直接触发
  • ❌ 多级CDN回退读取+合并 — 数据不一致，各设备不同步
  • ❌ GitHub Issues API — CORS可行但会污染Issues页面
  • ✅ Supabase 数据库 — 2个REST API调用搞定

最终方案由AI推荐、我评估可行性后实施。整条链路从"数据写不进GitHub"到"一个API写入数据库"，代码量从200行降到10行。

4. AI的局限性 & 人不可替代的部分

AI并非万能，以下场景仍需人工判断：

  • 架构选型 — AI可以提供方案对比，但技术选型需要理解业务场景和约束条件
  • 审美决策 — 贴图效果好不好、UI配色是否协调，最终得人说了算
  • 安全审查 — Token泄露风险、CORS配置、数据库权限，AI会给出方案但安全把关得自己来
  • 持续迭代方向 — "下一步做什么"这个问题的答案需要产品思维，AI只能辅助

🎯 结语

这次经历让我深刻理解到，AI不是编程的终点，而是编程的放大器。 它把"从0到1"的成本降到了极低，让一个人就能完成从前需要设计师、前端、后端、运维 四个角色才能做到的事情。

但同时，AI编程也提出了新的要求：你需要有判断力去评估AI的输出， 有产品思维去定义需求，有系统思维去设计架构。 AI不会取代程序员，但会用AI的程序员会取代不会用的。
